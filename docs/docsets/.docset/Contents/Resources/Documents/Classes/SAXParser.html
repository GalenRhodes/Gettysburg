  • structureSizeLimit

    For performance, SAXParser tries to read as much of an XML structure into memory and then analyze it as a whole. This setting limits the number of characters read at once to keep memory from being exhausted. If this limit is hit then an error is thrown. Normal text blocks, including CDATA sections are “chunked” if they exceed this limit so that no error is thrown. Except for text blocks, this limit should never be breached for a well formed document. Allowed range is 4KB <= limit <= 4MB. The default is 1MB.
